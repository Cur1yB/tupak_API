Tupak API

Алоха, Тупак! Это API моего проекта. Ты сможешь обращаться к нему из Android‑приложения: отправлять HTTP‑запросы и получать ответы в формате JSON. Сейчас в API реализован простой модуль Users — операции создания, получения, обновления и удаления пользователей.

Общее описание

  • Базовая сущность: пользователь (id, name, email).
  • Формат данных: JSON (Content-Type: application/json).
  • Хранилище: временное (in-memory), порядок распределения соответствует добавлению.
  • Валидация: проверяется формат данных, например email.
  • Уникальность: email должен быть уникальным (иначе ошибка 400).

Маршруты Users (CRUD)

  • GET /users — получить список всех пользователей (ответ 200).
  • POST /users — создать пользователя. Пример тела запроса: { "name": "...", "email": "..." }. Ответ 201 (ошибка 400, если email уже существует).
  • GET /users/{user_id} — получить пользователя по ID (200; если не найден — 404).
  • PUT /users/{user_id} — обновить пользователя по ID (name и/или email, если email занят — 400).
  • DELETE /users/{user_id} — удалить пользователя по ID (успех 204 No Content, если не найден — 404).

Все примеры запросов и возможности “Try it out” находятся в документации по адресу: SwaggerUI
